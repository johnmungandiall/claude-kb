# Design + Plan — claude-kb v2.2 feature batch

**Date:** 2026-06-26
**Status:** Approved (user: "add all, take over until complete"; feature #8 multilingual skipped)

Adds nine compact capabilities on top of v2.1. Each is implemented as additively
as possible to respect the repo's "keep it SMALL" philosophy. Prompt edits are
mirrored verbatim into `README.md`'s fenced blocks (sync gate: exact diff).

## Features

1. **Last-verified marker** — RULE in `prompt.md`: tag volatile `path:line` refs
   with `(checked <date>)` so staleness is visible.
2. **`kb/gotchas.md`** — known traps / footguns / "do NOT do X". New optional note.
3. **`verify.md`** — a third standalone prompt that audits the KB against the code
   and reports drift, without rebuilding. README gets a third fenced block; a
   `kb/features/prompt-verify.md` note documents it.
4. **Mermaid diagram** — STEP 3 `architecture.md` may include a tiny mermaid
   diagram when it clarifies the module/data flow.
5. **`kb/changelog.md`** — dated one-line history of notable KB changes; AUTO-MAINTAIN
   appends an entry when the KB changes.
6. **Global user prefs** — USER UNDERSTANDING rule extended: repo-specific prefs
   stay in `kb/about-you.md`; cross-project prefs also persist to host long-term
   memory (e.g. Claude Code memory) when available.
7. **Secret guard** — RULE: never copy secrets/keys/tokens/PII into the KB;
   reference the location, never the value.
8. ~~Multilingual KB~~ — **SKIPPED** per user.
9. **Size guard** — strengthen the ≤ 50-line rule: split an oversized note into
   sub-notes instead of letting it sprawl.
10. **`kb/cheatsheet.md`** — one-page run/build/test commands + key entry points.

## File-change map

- `prompt.md` — RULES (1, 7, 9), STEP 3 tree lines (2, 4, 5, 10), USER rule (6),
  AUTO-MAINTAIN changelog line (5), map line for the new notes.
- `update.md` — USER rule (6); new step to create gotchas/changelog/cheatsheet if
  useful; map line.
- `verify.md` — **new** standalone audit prompt (3).
- `README.md` — mirror all three prompt files into fenced blocks; add a "Verify
  prompt" section and a "What's new in v2.2" note.
- `kb/features/prompt-verify.md` — **new** feature note for `verify.md`.
- Dogfood: create `kb/gotchas.md`, `kb/changelog.md`, `kb/cheatsheet.md`;
  update `CLAUDE.md` (map + USER rule + verify.md in the Editing-the-prompts note),
  `kb/conventions.md`, `kb/glossary.md`, `kb/overview.md` (version → 2.2).

## Verification

- `grep` each insertion is present.
- README fenced blocks diff byte-identical to `prompt.md`, `update.md`, `verify.md`.
- Every new `kb/*.md` ≤ 50 lines.

## Out of scope (YAGNI)

- No automated mermaid generation tooling — it is a prompt instruction only.
- No real secret-scanner — a written rule, not code.
- Global memory is "persist if the host supports it", not a new storage system.
